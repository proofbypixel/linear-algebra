Proof by Pixel

Deriving the visual world from scratch using vectors, dot products, and gradients.

How do we teach a computer to see? Or conversely, how do we teach it to draw? Traditional rasterization draws triangles. But there is a more mathematical approach: Ray Marching. It turns rendering into a root-finding problem, solvable with simple linear algebra. 1

1. This technique is distinct from Ray Tracing. Instead of calculating ray-triangle intersections, we march along a ray using a distance field.

1. The World as a Vector Space

In this paradigm, we define the world using a Signed Distance Function (SDF). An SDF, let's call it $f(\mathbf{p})$, takes any vector position $\mathbf{p} \in \mathbb{R}^3$ and returns a scalar: the distance to the nearest surface.

$$ f: \mathbb{R}^3 \to \mathbb{R} $$

If $f(\mathbf{p}) < 0$, the vector is inside the object. If $f(\mathbf{p}) = 0$, it is on the surface. For a sphere at the origin with radius $r$, the math is simply the Euclidean norm of the vector:

$$ f(\mathbf{p}) = \|\mathbf{p}\| - r $$

Here is how we translate that linear algebra concept into code:

import numpy as np

def sphere_sdf(p, radius):
    # p is a numpy vector [x, y, z]
    # We use the L2 Norm (Euclidean distance)
    return np.linalg.norm(p) - radius

2. The Ray (Parametric Line)

To see the sphere, we shoot rays from a camera. A ray is defined by a parametric line equation. Given an origin vector $\mathbf{ro}$ and a normalized direction vector $\mathbf{rd}$:

$$ \mathbf{p}(t) = \mathbf{ro} + t \cdot \mathbf{rd} $$

Our goal is to find the smallest scalar $t > 0$ such that $f(\mathbf{p}(t)) = 0$. 2

2. Since $\mathbf{rd}$ is normalized ($||\mathbf{rd}|| = 1$), the scalar $t$ represents the exact distance traveled along the ray.

3. Lighting via Dot Products

Once our ray hits a surface at point $\mathbf{p}$, we need to calculate how light reflects off it. Lambertian reflectance states that the intensity of light is proportional to the cosine of the angle between the surface normal $\mathbf{n}$ and the light direction $\mathbf{l}$.

In linear algebra, the cosine of the angle between two normalized vectors is equivalent to their Dot Product:

$$ Intensity = \max(0, \mathbf{n} \cdot \mathbf{l}) $$

But where do we get the normal $\mathbf{n}$? In a mesh, normals are stored. In an SDF, the normal is the Gradient of the distance field ($\nabla f$). We approximate this gradient using finite differences:

$$ \mathbf{n} = \text{normalize}\left( \nabla f(\mathbf{p}) \right) \approx \text{normalize}\begin{bmatrix} f(p_x+\epsilon) - f(p_x-\epsilon) \\ f(p_y+\epsilon) - f(p_y-\epsilon) \\ f(p_z+\epsilon) - f(p_z-\epsilon) \end{bmatrix} $$ 
def get_normal(p):
    e = 0.001
    d = scene_sdf(p)
    # The gradient points in the direction of greatest increase (away from surface)
    gradient = np.array([
        scene_sdf(p + [e, 0, 0]) - d,
        scene_sdf(p + [0, e, 0]) - d,
        scene_sdf(p + [0, 0, e]) - d
    ])
    return gradient / np.linalg.norm(gradient)

By strictly applying operations on vectors (norms, dot products, cross products) and scalars, we render a 3D image without ever defining a polygon.

We take a physical system $S$ which can occupy distinct states like $e_1, e_2, \dots$. The rule for movement is simple but powerful: The system has no memory.

Read Log 002 →

I was reading Markov Chains from the book Probability Theory: A Concise Course by Y.A. Rozanov.

1. Introduction:
Here we take a physical system $S$ which can occupy distinct states like $e_1, e_2, \dots$. Starting from an initial state when time $t=0$, the system moves from one state to another at each tick of the clock. This collection of all possible states is called the State Space.

2. The Rule (No Memory):
The rule for movement is: The system has no memory. The probability of the system occupying state $j$ at the next time step $n+1$, given the entire history of previous states $X_0$ to $X_n$, is equal to the probability dependent only on the current state $X_n$. $$P(X_{n+1} = j | X_0, \dots, X_n = i) = P(X_{n+1} = j | X_n = i)$$ This specific kind of memoryless randomness is called the Markov Property.

3. The Matrix (Data Science Application):
In Data Science, we don't just see this as abstract math; we see it as a transition map. Imagine modeling Customer Churn. State 1 is "Subscribed" and State 2 is "Unsubscribed". The matrix $P$ holds the probabilities of a user switching between these states next month. $$\sum_{j} p_{ij} = 1$$ 

import numpy as np
# A Churn Matrix: Row 0 (Active), Row 1 (Churned)
P = np.array([[0.90, 0.10], [0.05, 0.95]])
print(P.sum(axis=1)) # Output: [1. 1.]


II. Here's where it got very interesting

Predicting the future... If $P$ tells us where we go in one step, can we predict where we will be in the distant future?

Suppose we have already taken $n$ steps (let's say $n=2$ days). We want to know the probability of moving from state $i$ to state $j$ after another $m$ steps (say $m=3$ more days). The logic is simple: To get from $i$ to $j$ in total time ($n+m$), we just multiply the probability of the first leg of the trip ($n$) by the probability of the second leg ($m$). $$p_{ij}^{(n+m)} = \sum_{k} p_{ik}^{(n)} p_{kj}^{(m)}$$ These are the famous Chapman-Kolmogorov Equations.

# Predicting 5 days into the future (n=2 + m=3)
P_2 = np.linalg.matrix_power(P, 2)
P_3 = np.linalg.matrix_power(P, 3)
P_5 = np.dot(P_2, P_3)

I. The Theory

Started with vectors, the theory.

1. Pretty abstract to start with:
Vectors are elements of a vector space $V$. There can’t be a vector in isolation; there has to be a structure around it.

2. What is a vector space?
A Vector Space is a set $V$ combined with a Field $\mathbb{F}$ (usually real numbers, $\mathbb{R}$) equipped with two fundamental operations:

  • Vector Addition: $\mathbf{u} + \mathbf{v} \in V$
    (If you add two vectors, you get a vector).
  • Scalar Multiplication: $c \cdot \mathbf{v} \in V$
    (If you scale a vector, you get a vector).

3. The Axioms:
A set is a vector space if and only if it satisfies 8 specific rules, such as Associativity, Commutativity, and the existence of an Identity Element ($\mathbf{0}$).

4. Functions as Vectors:
Since functions also obey these rules (you can add $f(x) + g(x)$), mathematically, they are vectors too. This is the basis of Functional Analysis.

5. In the Euclidean space:
In Data Science, we usually work in $\mathbb{R}^n$. Here, a vector $\mathbf{v}$ is an $n$-tuple of real numbers: $$ \mathbf{v} = \begin{bmatrix} v_1 \\ v_2 \\ \vdots \\ v_n \end{bmatrix} $$

6. Geometrically (The Tensor View):
Vectors are defined by how they handle coordinate transformations. It’s intuitive to understand this with the transformation rule using the Jacobian Matrix. A vector must transform against the change of basis (Contravariance): $$ v'^i = \sum_{j=1}^n \frac{\partial x'^i}{\partial x^j} v^j $$



II. Understanding vectors with python

Understanding vectors using Python.

1. Vector Addition: Using NumPy

import numpy as np
a = np.array([1, 2])
b = np.array([3, 4])
print(f"a + b = {a + b}")
# Output: [4 6]

It stays in $\mathbb{R}^2$ and even $b + a$ gives the same result, hence commutativity holds.

2. Visual Proof: Breaking down the Matplotlib Code

To prove that [4, 6] is actually the result of "walking" along the vectors, we visualized it. Here is the breakdown of the code we used to generate the plot.

Step A: The Setup
First, we define a helper function to draw arrows. The function uses plt.quiver, which is designed for vector fields.

def plot_arrow(start, vector, color, label):
    plt.quiver(*start, *vector, angles='xy', scale_units='xy', scale=1, color=color, label=label)

Step B: The Plotting
We plot the first vector ($\vec{a}$), and then—this is the trick—we plot the second vector ($\vec{b}$) starting from the tip of $\vec{a}$.

origin = [0, 0]
# 1. The First Step (Blue)
plot_arrow(origin, a, 'blue', 'a = [1, 2]')
# 2. The Second Step (Red) - Notice the start point is 'a'
plot_arrow(a, b, 'red', 'b = [3, 4]')
# 3. The Result (Green) - From Origin to Destination
result = a + b
plot_arrow(origin, result, 'green', 'Result = [4, 6]')

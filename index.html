Concept Summary (With Math + Python Awareness)

1. Norm (Length)

A norm assigns a non-negative length to a vector. Formally, a norm $\|\cdot\|$ on $V$ satisfies the triangle inequality and scaling rules.

The most familiar is the Euclidean norm:

$$ \|\vec{v}\|_2 = \sqrt{\vec{v}^\top \vec{v}} $$

Python Awareness: Vector length is computed directly.

length = np.linalg.norm(v)

2. Distance (Derived from Norm)

Once a norm exists, it defines a distance. Distance isn‚Äôt a new concept‚Äîit‚Äôs just the length of the difference vector.

$$ d(\vec{u}, \vec{v}) = \|\vec{u} - \vec{v}\| $$

Python Awareness: Distance is the norm of the difference.

dist = np.linalg.norm(u - v)

3. Inner Product & Angles

An inner product introduces geometry (angles) into a vector space. It allows us to define Orthogonality.

$$ \cos(\theta) = \frac{\langle \vec{u}, \vec{v} \rangle}{\|\vec{u}\| \|\vec{v}\|} $$

If $\langle \vec{u}, \vec{v} \rangle = 0$, the vectors are orthogonal (perpendicular). Zero interaction means zero alignment.

Python Awareness: Inner products are dot products.

# Check for orthogonality (approx 0)
is_orthogonal = np.dot(u, v) == 0

Figure 1: The geometry of two vectors. The green line is the distance; $\theta$ determines the inner product.

4. Common Norms (Quick Contrast)

  • $\ell_2$ (Euclidean): Smooth, geometric distance.
  • $\ell_1$ (Manhattan): Sum of absolute values ($\sum |v_i|$).
  • $\ell_\infty$ (Max): Maximum magnitude ($\max |v_i|$).

How This Connects to Data Science

These concepts show up quietly but everywhere in ML.

1. Similarity vs. Distance
Two vectors can be close in distance but poorly aligned‚Äîor perfectly aligned but far apart.
Example: Cosine similarity uses the Inner Product (ignoring magnitude), while Euclidean distance uses the Norm (sensitive to magnitude).

2. Regularization Intuition
Regularization terms like $\lambda \|\vec{w}\|_2^2$ penalize large parameter norms. Geometrically, this constrains the solution to lie inside a specific shape (a ball for $\ell_2$, a diamond for $\ell_1$). This geometric constraint forces the model to be simpler.

Concept Summary (With Math + Python Awareness)

1. Rank

The rank of a matrix is the dimension of the space spanned by its columns. Rank tells us how many linearly independent directions actually exist in the data, regardless of how many columns the matrix has.

$$ \text{rank}(A) = \dim(\text{Col}(A)) $$

Python Awareness: Rank measures real information, not just dimensions.

# If features are redundant, rank < number of columns
real_info = np.linalg.matrix_rank(X)

2. Linear Mapping ($T$)

A function $T: V \to W$ is linear if it preserves structure (respects addition and scaling). It does not bend or shift space; it only rotates, stretches, or shears it.

$$ T(\vec{x}) = A\vec{x} $$

Python Awareness: Linear mappings are just matrix multiplication.

output = A @ x

3. Kernel & Image (The Anchors)

  • Kernel (Null Space): Inputs that get collapsed to zero ($\vec{x}$ where $A\vec{x} = \vec{0}$).
  • Image (Range): Everything the mapping can produce.

Rank-Nullity Insight: $\dim(V) = \text{rank}(T) + \dim(\ker(T))$. Information is either kept or killed.

4. Affine Spaces

An affine space is a vector space that has been shifted away from the origin. $$ \mathcal{A} = \vec{v}_0 + U $$ where $U$ is a subspace and $\vec{v}_0$ is a fixed offset.

Python Awareness: Affine transformations add a shift vector.

# The "+ b" makes it Affine
y = A @ x + b

How This Connects to Data Science

I‚Äôm approaching this from reading rather than experience, but these concepts explain model capacity and bias.

1. Rank and Model Capacity
If a dataset has 10 features but the feature matrix has a Rank of 3, the model effectively operates in 3 dimensions. This explains why adding more features sometimes does nothing‚Äîif they are dependent, the Rank doesn't increase, and the model doesn't learn anything new.

2. Linear vs. Affine Models
A pure Linear model forces predictions to pass through the origin $(0,0)$. An Affine model allows a shift via an intercept (bias term).


Figure 1: Linear transformations are tethered to the origin. Affine spaces are free to move.
  • $y = A\vec{x}$ (Linear: Must hit 0)
  • $y = A\vec{x} + \vec{b}$ (Affine: Can shift)

That small $\vec{b}$ term changes the geometry entirely, allowing the model to fit data that doesn't center at zero.

Concept Summary (With Math + Python Awareness)

1. Vector Space

A vector space $V$ over a field $\mathbb{F}$ is a set where vector addition and scalar multiplication are defined and satisfy specific axioms. Formally, for all $\vec{u}, \vec{v} \in V$ and $\alpha \in \mathbb{F}$:

$$ \vec{u} + \vec{v} \in V \quad \text{and} \quad \alpha \vec{v} \in V $$

Python Awareness: Vector spaces show up as arrays supporting addition and scaling.

v = np.array([1, 2, 3])
# Result remains in the same "space" (array of size 3)
result = 2 * v + v

2. Subspace

A subspace $U \subseteq V$ is a subset that is itself a vector space. Instead of checking all axioms, we verify three conditions:

  1. $\vec{0} \in U$ (Contains the zero vector).
  2. If $\vec{u}, \vec{v} \in U$, then $\vec{u} + \vec{v} \in U$ (Closed under addition).
  3. If $\vec{v} \in U$ and $\alpha \in \mathbb{F}$, then $\alpha \vec{v} \in U$ (Closed under scaling).

Python Awareness: Subspaces appear when linear combinations ($a \cdot u + b \cdot v$) stay within the same space.

3. Span

Given vectors $\vec{v}_1, \dots, \vec{v}_n$, their span is the set of all reachable vectors:

$$ \text{span}(\vec{v}_1, \dots, \vec{v}_n) = \{ \sum_{i=1}^{n} \alpha_i \vec{v}_i \mid \alpha_i \in \mathbb{F} \} $$

I‚Äôm starting to think of span as the boundary of possibility‚Äîanything outside it is unreachable without new vectors.

Python Awareness: Span corresponds to the column space of a matrix.

# Checking the rank (dimension of the span)
rank = np.linalg.matrix_rank(X)

4. Linear Independence

A set of vectors is linearly independent if the only way to sum them to zero is by setting all coefficients to zero:

$$ \sum \alpha_i \vec{v}_i = \vec{0} \implies \alpha_1 = \dots = \alpha_n = 0 $$

Reframed: If one vector can be built from the others, it isn‚Äôt adding a new direction. Independent vectors expand the span; dependent ones don‚Äôt.

Python Awareness: Independence shows up as full column rank.

# If True, columns are independent
is_independent = np.linalg.matrix_rank(X) == X.shape[1]

How This Connects to Data Science

I‚Äôm not a data scientist, but while reading about regression and feature engineering, these ideas kept showing up‚Äîespecially around multicollinearity. Once I learned that features are treated as vectors, the connection became clear.

Example: Redundant Features
Imagine a dataset with two columns:

  • $\vec{v}_1$: Age in years $\rightarrow [20, 30, 40]$
  • $\vec{v}_2$: Age in months $\rightarrow [240, 360, 480]$

Since $\vec{v}_2 = 12 \cdot \vec{v}_1$, these vectors are Linearly Dependent.

  • Linear Algebra View: Both generate the same span.
  • Machine Learning View: One feature adds no new information.

Figure 1: Linearly Dependent vectors lie on the exact same trajectory.

This is exactly what causes Multicollinearity‚Äîfeatures that look different but encode the same direction, which can destabilize regression models.
